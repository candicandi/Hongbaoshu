# 发布检查清单

每次发布新版本前,请按照此清单进行检查,以减少用户遇到安全警告的情况。

## 构建前检查

- [ ] 更新 `versionCode` 和 `versionName` (在 `app/build.gradle.kts`)
- [ ] 检查 `keystore.properties` 配置正确
- [ ] 确认 `release.jks` 密钥库文件存在
- [ ] 更新 `CHANGELOG.md` 或 Release Notes

## 构建 Release APK

```bash
# 清理旧的构建文件
./gradlew clean

# 构建 release APK
./gradlew assembleRelease

# APK 位置
# app/build/outputs/apk/release/app-release.apk
```

## 构建后验证

### 1. 验证签名

```bash
# 查看 APK 签名信息
apksigner verify --print-certs app/build/outputs/apk/release/app-release.apk

# 或使用 keytool
keytool -printcert -jarfile app/build/outputs/apk/release/app-release.apk
```

**记录以下信息**:
- [ ] SHA-256 指纹
- [ ] 签名有效期
- [ ] 签名者信息

### 2. 测试安装

- [ ] 在真机上测试安装
- [ ] 验证应用功能正常
- [ ] 检查应用大小是否合理
- [ ] 测试升级流程(如果是更新版本)

### 3. 安全扫描

#### VirusTotal 扫描

1. 访问 https://www.virustotal.com/
2. 上传 `app-release.apk`
3. 等待扫描完成(约 2-5 分钟)
4. 记录扫描结果链接

- [ ] VirusTotal 扫描完成
- [ ] 记录扫描结果 URL
- [ ] 检查误报情况(理想情况下应该是 0/60+ 或极少数误报)

#### 本地安全软件测试(可选)

如果可能,在安装了以下安全软件的设备上测试:
- [ ] 360 手机卫士
- [ ] 腾讯手机管家
- [ ] 华为/小米/OPPO 等厂商内置安全中心

## 准备发布资料

### 1. 更新文档

- [ ] 更新 `README.md` 中的版本信息
- [ ] 更新 `docs/安装指南.md` 中的签名指纹信息
- [ ] 准备 Release Notes(中英文)

```markdown

- 功能 1
- 功能 2

### 改进
- 改进 1
- 改进 2

### 修复
- 修复 1
- 修复 2

---

## 安全信息

### APK 签名指纹
```
SHA-256: [从 apksigner 获取的指纹]
```

### VirusTotal 扫描
- 扫描结果: [扫描结果链接]
- 检测率: 0/XX (理想情况)

---

## 下载
- [app-release.apk](链接)
- 文件大小: XX MB
- 最低 Android 版本: 9.0 (API 28)
```

### 3. 发布检查

- [ ] 上传 APK 到 GitHub Release
- [ ] 填写完整的 Release Notes
- [ ] 包含 VirusTotal 扫描链接
- [ ] 包含签名指纹信息
- [ ] 添加安装指南链接
- [ ] 设置正确的 tag (如 v1.0.4)

## 发布后

### 1. 验证下载

- [ ] 从 GitHub Release 页面下载 APK
- [ ] 验证下载的文件 SHA-256 与本地构建一致
- [ ] 测试安装下载的 APK

### 2. 监控反馈

- [ ] 关注 GitHub Issues 中的安装问题反馈
- [ ] 记录用户报告的安全软件误报情况
- [ ] 准备常见问题解答

### 3. 更新记录

在项目中记录:
- [ ] 本次发布的 VirusTotal 扫描结果
- [ ] 用户报告的误报情况
- [ ] 不同安全软件的表现

## 减少误报的最佳实践

### 已实施
- ✅ 使用正式的发布签名
- ✅ 启用 ProGuard 代码混淆
- ✅ 启用资源压缩
- ✅ 提供详细的安装指南
- ✅ 每次发布进行 VirusTotal 扫描

### 可选优化
- ⭕ 考虑购买代码签名证书($100-300/年)
- ⭕ 提交到 F-Droid 开源应用商店
- ⭕ 建立官方网站提供下载和验证工具
- ⭕ 联系主要安全软件厂商申请白名单

---

## 紧急情况处理

如果发布后发现严重问题:

1. **立即标记 Release 为 Pre-release**
2. **在 Release 页面添加警告说明**
3. **准备修复版本**
4. **通知已下载的用户(通过 Issues/Discussions)**

## 附录: 有用的命令

### 查看 APK 信息
```bash
# APK 分析
aapt dump badging app-release.apk

# 查看签名
apksigner verify --verbose app-release.apk

# 计算文件哈希
shasum -a 256 app-release.apk
```

### 生成新的密钥库(如需要)
```bash
keytool -genkeypair -v \
  -keystore hongbaoshu-release.jks \
  -keyalg RSA \
  -keysize 2048 \
  -validity 10000 \
  -alias hongbaoshu-release
```
