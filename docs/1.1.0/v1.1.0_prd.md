# 用户反馈
## 优先级0
- 很多人还是不知道有功能菜单
- 很多人不知道有目录功能
- 希望能方便的播放、暂停
- 用户反馈后台播放功能无法生效、或者播放一会就停了，系统音频控制台也看不到正在播放的音频


# 现状梳理（结合当前实现）
- 菜单入口为“屏幕顶部 20% 区域双击”唤出，下拉菜单内包含目录入口与听书设置
- 新手引导弹窗仅提示一次“双击顶部区域可唤出菜单”，用户可选择不再提醒
- 目录以对话框形式展示，仅在菜单里通过“目录”按钮进入
- 朗读模式为开关，开启后自动播放当前页第一句，语速可调；缺少显式“播放/暂停”按钮
- 音频播放采用本地播放组件，未建立前台服务/媒体会话，系统级媒体控制台不一定能识别

# 问题归因
## 入口发现性不足
- 功能菜单需要学习成本（双击顶部区域），且入口无持久化视觉提示
- 目录入口隐藏在菜单内，用户不知道菜单则无法触达目录

## 播放/暂停操作不直观
- 朗读模式当前以开关方式呈现，用户难以将“朗读开关”理解为“播放/暂停”
- 已存在暂停/恢复能力，但未暴露在界面上

## 后台播放稳定性与系统集成不足
- 播放未提升为前台服务，系统可能在后台回收导致播放中断
- 未接入系统媒体会话，导致系统音频控制台无法显示或控制

 
# 竞品交互调研（要点）
## 微信读书
- 阅读页通过单击中间区域唤出上/下工具栏与悬浮入口，属于“阅读时唤出工具栏”的模式（实测补充）
- 底部工具栏包含进度、显示设置、字体设置与目录入口等核心操作（微信读书技巧总结）
- 听书/朗读提供倍速与定时等控制，并允许关闭背景音，偏向“专门听书页面集中操作”的思路（微信读书听书体验）

## 亚马逊阅读器（阅读器与应用通用习惯）
- 阅读时通过点击顶部或中部唤出工具栏，工具栏并非常驻，强调沉浸式阅读与“用时即显、用完即隐”的交互（亚马逊帮助、阅读器指南）


# 改进计划
## 优先级0（本次版本必须完成）
- 增加显式的“播放/暂停”控件：随阅读工具栏浮现，与朗读状态同步，默认不常驻
- 菜单入口可见化：阅读页采用“轻触唤出工具栏 + 首次轻提示”的组合，降低学习成本
- 目录入口直达：目录入口跟随工具栏出现，弱化对手势入口的依赖
- 建立前台播放能力：引入前台服务与媒体会话，确保后台播放稳定并显示系统媒体控制

## 功能设计
### 工具栏与入口
- 触发方式：单击中间区域唤出上工具栏与下工具栏，3-5 秒无操作自动隐藏
- 上工具栏：返回、书名/章节、更多入口
- 下工具栏：进度条、目录入口、朗读入口、显示设置入口
- 悬浮入口：仅在工具栏收起后保留“小圆点”提示，10 秒后自动淡出

### 工具栏组件与字段
- 工具栏状态：是否可见、上次交互时间、自动隐藏时长、交互中标识
- 顶部栏：标题、章节标题、返回、更多入口
- 底部栏：进度、进度拖动、打开目录、打开朗读设置、打开显示设置、切换朗读
- 悬浮提示：是否可见、淡出时长、点击打开工具栏

### 目录与定位
- 目录弹窗支持章节列表与当前章节高亮
- 进入目录后，顶部保留搜索入口用于书内搜索
- 支持快速回到当前阅读位置

### 朗读与播放控制
- 朗读入口进入“播放控制浮层”，包含播放/暂停、上一句/下一句、倍速、定时关闭
- 播放状态与系统媒体控制台同步，支持耳机线控暂停/继续
- 朗读异常时提示“切换前台或重新播放”并提供一键重试

### 朗读状态与行为定义
- 状态：关闭 / 加载中 / 播放中 / 暂停 / 异常
- 进入条件：点击朗读入口 → 加载中 → 播放中
- 暂停条件：点击暂停或耳机线控暂停 → 暂停
- 恢复条件：点击播放或耳机线控播放 → 播放中
- 异常处理：播放失败提示并记录错误码，提供重试与返回阅读

### 后台播放
- 前台服务在朗读或背景音乐启动时启用，停止后自动释放
- 锁屏通知提供播放/暂停与下一句

### 前台服务与媒体会话
- 启动：朗读/播放开始时创建前台服务与媒体会话
- 更新：播放状态变化同步更新通知与媒体会话元数据
- 结束：朗读停止或退出阅读页时释放前台服务

### 进度条与章节跳转
- 进度条支持拖动定位，拖动时显示目标章节与页码
- 释放后自动跳转并关闭工具栏，避免遮挡
- 章节跳转支持“上一章/下一章”快捷按钮

### 进度与目录数据
- 进度范围：0-100，映射到章节与页码
- 当前章节序号、当前页序号、总页数
- 目录数据：章节标题、章节序号、页数统计

### 设置项细化
- 显示设置包含亮度、背景色、字体大小与行距
- 朗读设置包含语速与定时（15/30/60 分钟与读完本章）

### 朗读浮层控件规格
- 播放/暂停按钮：48 密度像素，主色高亮
- 上一句/下一句：32 密度像素，次级色
- 倍速选择：0.8、1.0、1.2、1.5
- 定时关闭：15/30/60 分钟、读完本章、关闭定时

## 交互设计
### 触发与隐藏
- 单击中间区域唤出工具栏，点击空白区域或 3-5 秒无操作自动隐藏
- 翻页与朗读播放时默认不弹出工具栏，避免干扰阅读

### 轻提示策略
- 首次进入阅读页显示“单击中间区域唤出工具栏”轻提示，3 秒后淡出
- 提示提供“本次不再显示”，不影响后续体验

### 目录交互细节
- 目录弹窗打开后支持滑动定位，点击章节后自动关闭并跳转
- 支持“返回当前进度”快捷入口

### 朗读交互细节
- 播放/暂停按钮在播放中保持醒目，但仅在工具栏唤出时出现
- 朗读状态与页面高亮同步，暂停时保留高亮

### 工具栏状态机
- 状态：隐藏 / 显示 / 操作中 / 自动隐藏倒计时
- 触发：单击中间区域进入显示，用户操作进入操作中
- 自动隐藏：无操作 3-5 秒进入隐藏，播放/拖动时不触发隐藏

### 事件与交互埋点
- 工具栏展示事件：触发方式（中部轻触/悬浮提示）
- 目录打开事件：来源（底部工具栏/快捷入口）
- 朗读切换事件：状态（播放/暂停/停止）
- 进度拖动事件：起止位置与耗时

### 关键流程
- 打开工具栏：单击中间区域 → 工具栏显示 → 3-5 秒自动隐藏
- 播放/暂停：点击播放 → 系统媒体控制台显示 → 再次点击暂停
- 目录跳转：打开目录 → 选择章节 → 跳转 → 工具栏收起
- 进度拖动：拖动进度条 → 显示目标章节/页码 → 释放跳转

## 可落地的任务拆分
- 新增阅读工具栏组件与状态管理
- 新增朗读控制浮层与定时逻辑
- 引入前台服务与媒体会话同步播放状态
- 调整目录弹窗为带搜索与进度回跳的版本

## 交互原则（参考主流阅读应用习惯）
- 阅读页保持“无控件常驻”，操作入口以轻提示或手势触发为主
- 控件只在用户主动操作后短时浮现，默认 3-5 秒自动隐藏
- 核心操作优先使用顶部/底部轻量工具栏，避免悬浮控件长驻遮挡正文
- 目录与播放控制应跟随工具栏一起出现与消失，降低视觉干扰
- 所有提示性控件需支持“本次不再显示”，避免反复打扰

## 优先级1（提升体验）
- 新手引导优化：结合“首次进入/首次使用朗读/首次打开菜单”多节点提示
- 菜单发现性强化：首次进入阅读页显示短时浮层提示“顶部双击/点击按钮”
- 播放异常自检：捕获播放错误并提示用户恢复建议（例如切换到前台重试）

## 设计与验收要点
- 任何播放状态在系统通知与锁屏控制台可见，并可暂停/恢复
- 菜单与目录入口在 3 秒内可被新用户发现
- 朗读可在后台持续播放不少于 5 分钟且不中断
- 阅读页在无操作状态下全屏仅正文，不出现常驻控件
