# 技巧
- 一次只解决一个问题
- 连续多次解决不了
    - 让模型尝试换实现方案，而不是只盯着当前方案打补丁
    - 换模型
    - 新建一个上下文窗口重来一次，减少之前的上下文干扰
- 让AI给你讲一下某一个地方的设计逻辑，这可能与 prd 有出入，或者 prd 没有讲清楚，AI 自行设计了，这些设计可能是不合理、不优雅的，挑战这些逻辑，重新设计；AI 再输出这些逻辑的过程中可能也会顺便发现问题所在。

- [fixed]再优化一下页面展示逻辑
保持现在的分页逻辑不变，原来的方案是上下 10% 留白，中间 80% 作为文字容器； 现在改成上部 10% 留白，中间 80% 容器，然后再给 3% 作为缓冲区，最底部的 7% 留白；这个缓冲区不可以遮挡容器，缓冲区是为了避免容器有计算误差导致的极少情况下文字被留白区域给遮盖住了的情况；



- [fixed]优化分页逻辑
1. 目前的分页逻辑下，底部会发生部分文字溢出被遮挡的情况
2. 容器填充要按字符级别填充，最小单位不能是句子，也不能把原来的段落拆分出新段落， 无论是 annotation 还是正文，都保持这个按字符摆放的逻辑；annotation 在跨页的时候注意也要保持注释样式一致；
3. 不允许估算容器尺寸，必须用实际容器的高度来计算，保持渲染的精确性，不允许在容器边缘出现字符溢出、遮挡情况
4. 只允许在每章的最后的文字所在的页面底部允许出现空白，其他页面都要把容器填满（但不允许溢出、遮挡）
5. 以上改动同时要考虑朗读和高亮功能要兼容这种逻辑；
6. 避免影响翻页逻辑、二次菜单功能



-  [closed]优化分页逻辑
当前的分页逻辑下，每页存在大量空白或者文字溢出，检查当前的分页逻辑是怎么造成这种问题的，必要的情况下进行重构，保持方法的优雅、简洁，不要打补丁、不要过度设计；我的功能要求是要除了上下留出 10% 的留白，中间的 80% 都要填充文字，既不要溢出，也不要留出空白
